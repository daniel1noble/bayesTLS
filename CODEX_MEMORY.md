- Target journal: Methods in Ecology and Evolution.
- Current empirical focus: brown shrimp (`Crangon crangon`) lethal TDT,
  sublethal time-to-death, dynamic CTmax, and fluctuating-temperature injury
  projections.
- Older / prototype focus still present in docs and code: zebrafish survival
  and life-stage TDT. Treat older zebrafish references as historical unless the
  user explicitly asks to revive them.

Core scientific pitch:

The classical thermal death time / thermal load sensitivity pipeline estimates
per-temperature LT50 point estimates and then fits OLS to `log10(LT50) ~ T`.
This loses uncertainty, mishandles proportion/count data, drops or caps censored
temperatures, ignores overdispersion and hierarchy, and hides many analyst
degrees of freedom. The proposed replacement is a single Bayesian hierarchical
four-parameter logistic model fit to raw survival counts. Classical outputs
such as z, CTmax at a reference duration, LT curves, and heat-injury trajectories
are derived from posterior draws rather than estimated in a second stage.

## Main Statistical Model

The core 4PL survival model is:

```text
p_surv(t, T) = low + (up - low) / (1 + exp(k * (log10(t) - mid(T))))
```

where:

- `low` is the lower survival asymptote.
- `up` is the upper survival asymptote.
- `k = exp(logk)` is the positive steepness on the log10-duration axis.
- `mid(T) = beta_0 + beta_1 * (T - T_bar) + ...` is the temperature-dependent
  midpoint on the log10-duration axis.

Identifiability parameterisation used in the older shrimp notebook:

```text
low = inv_logit(lraw) * 0.49
up  = 0.51 + inv_logit(uraw) * 0.49
k   = exp(logk)
```

Newer reusable-function notebook uses safer non-boundary bounds:

```text
low = 0.001 + inv_logit(lraw) * 0.498
up  = 0.501 + inv_logit(uraw) * 0.498
```

Why these bounds matter:

- Exactly 0 or 1 creates `logit(0)` or `logit(1)`, which can cause Stan
  initialisation failures.
- Keeping asymptotes strictly inside (0, 1) prevents `log probability evaluates
  to log(0)` at initialisation.

The classical mapping:

For target survival `x`, solving the 4PL for duration gives:

```text
log10(LTx(T)) = beta_0 + beta_1 * (T - T_bar)
              + (1 / k) * log((up - x) / (x - low))
```

For a one-group model with shape parameters not varying with temperature:

- `beta_1` is the TDT slope on `log10(time) ~ temperature`.
- `z = -1 / beta_1` in deg C per decade of time.
- The asymmetry term shifts the intercept but not the slope.
- CTmax at reference time `t_ref` is the temperature where the posterior LT
  curve crosses `t_ref`.

The newer functions usually derive z by numerically inverting posterior LT50
curves and fitting `log10(time) ~ temperature` per draw. This is deliberately
more general than only using `-1 / b_mid_temp_c`, because it also works for
grouped models, covariate-conditioned predictions, and non-50% thresholds.

## Key Code: R/bayesian_TDT_shrimp.qmd

This is the broad prototype analysis for brown shrimp. It is useful for
methodological ideas and CTmax/sublethal work, but some chunks are rough and
session-dependent.

Major sections:

- Lethal TDT:
  - Reads `LETHAL_TDT`.
  - Converts mortality proportions to `n_surv`, `n_dead`, `survival`.
  - Fits a binomial 4PL with OLRE:
    `mid ~ temp_c + (1 | date_experiment) + (1 | tank_ID)`,
    `z ~ 0 + (1 | obs)`.
  - Derives survival curves, TDT curves, 1 h survival temperatures, and 2D
    thermal landscapes.
- Lethal model variants:
  - Interaction model.
  - Body-mass model.
  - Starting-time model.
  - LOO/reloo comparisons. Comments say body mass, starting time, and
    interaction do not improve fit.
- Sublethal TDT:
  - Reads `SUBLETHAL_TDT` and trial data.
  - Fits `log10_time_to_death ~ temp_c + (1 | date_experiment) +
    (1 | tank_ID) + (1 | cup_ID)`.
  - Comments report z around 1.10.
  - Tests mortality rate and body mass; comments say the simple model is best.
  - Notes start time and temperature are highly collinear (`r = 0.71`).
- CTmax:
  - Reads fast ramping data and slow CTM files.
  - Computes ramping rates for approximately 0.02, 0.3, and 0.5 deg C/min.
  - Fits `CTmax ~ Ramping + (1 | Trial_ID) + (1 | Date_experiment)`.
  - Tests start time, body mass, ramping deviation, mortality, observer, and
    starting temperature; comments generally retain the parsimonious base model.
- Static-dynamic reconciliation:
  - Uses the Jorgensen dynamic CTmax equation.
  - Forward branch: sublethal TDT posterior z -> dynamic CTmax; estimate
    `sCTmax` per draw.
  - Inverse branch: dynamic CTmax -> TDT; estimate z and `sCTmax` per draw via
    `nls`.
  - Comments say static and dynamic z are not statistically different, though
    dynamic-derived z is lower on average.
- Field temperatures and heat injury:
  - Reads Kberg field temperature CSVs.
  - Filters study period around 2025-06-12 to 2025-06-30.
  - Integrates injury under current and +1/+2/+3 deg C warming scenarios.
- Repair model:
  - Uses Sharpe-Schoolfield repair kernels.
  - Damage is estimated from the lethal 4PL.
  - Repair parameters are fixed scenario assumptions, not fitted.

## Things Future Codex Should Not Forget

- The current polished lethal model is beta-binomial in `R/test_functions.qmd`;
  the older shrimp notebook is binomial + OLRE.
- Durations in the shrimp data are hours; many plots/tables convert to minutes.
- `z = -1 / b_mid_temp_c` is exact only for the simple one-group, fixed-shape
  case. Use posterior LT curve derivation for grouped/general models.
- Repair parameters are scenario assumptions, not estimated from the static TDT
  data.
- CTmax-like reference temperatures are extrapolations or near-edge quantities;
  credible intervals expose uncertainty but do not make extrapolation safe.
- The manifesto is intentionally sharp; the literature note says where to
  soften claims.
- Do not overwrite raw data.
- Do not revert the user's existing modified/deleted/generated files without
  explicit permission.
