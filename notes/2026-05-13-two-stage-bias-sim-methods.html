Methods section for a controlled simulation that compares three TDT estimators side-by-side: the field-default two-stage pipeline (binomial GLM Stage 1), a patched two-stage pipeline (beta-binomial GLM Stage 1) that addresses likelihood misspecification without changing the architecture, and the joint Bayesian 4PL developed in this paper.

1 Estimators

All three estimators are fit to every simulated dataset.

1.1 Two-stage (binomial) — the field default

Stage 1. For each temperature independently, a binomial GLM with logit link is fitted to survivor counts as a function of \(\log_{10} t\). The per-temperature \(\log_{10} \text{LT}_{50}\) is read off as \(-\hat\beta_0/\hat\beta_1\) with delta-method standard error from the Stage-1 covariance.

Stage 2. An unweighted ordinary least squares regression of the five \(\log_{10} \text{LT}_{50}\) values on temperature gives slope and intercept estimates from which \(z = -1/\hat\beta_1^{(2)}\) and \(\text{CT}_{\max,1\text{h}} = (\log_{10} 60 - \hat\beta_0^{(2)}) / \hat\beta_1^{(2)}\) follow. The 95% confidence interval comes from the delta method on the Stage-2 covariance. A dataset is marked as a two-stage success only when every temperature delivers a finite, negative-slope Stage-1 fit and the Stage-2 slope is itself finite and negative.

Unweighted Stage-2 OLS is retained deliberately — it matches the version of the pipeline most commonly reported in the empirical TDT literature and is the comparator the reader will be familiar with.

1.2 Two-stage (beta-binomial) — the patched variant

Same two-stage architecture, but Stage 1 swaps the binomial GLM for a beta-binomial GLM (logit link) fitted via glmmTMB::glmmTMB(..., family = betabinomial()). The dispersion parameter is estimated per temperature. Stage 2 remains unweighted OLS on the resulting per-T \(\log_{10} \text{LT}_{50}\) estimates.

This variant exists for one reason: to disentangle the likelihood component of the field default’s deficit from the shape / architecture component. The contrast between the binomial two-stage and the patched two-stage isolates the likelihood effect (both have the same two-stage architecture; only the Stage-1 likelihood differs). The contrast between the patched two-stage and the joint 4PL isolates the shape / architecture effect (both have the correct likelihood for the beta-binomial scenarios; only the architecture differs).

1.3 Joint Bayesian 4PL

The joint model is the estimator developed in this paper. It is a non-linear brms formula with a beta_binomial(link = "identity") family in which all four 4PL sub-parameters — the reparameterised lower asymptote, upper asymptote, slope, and midpoint — carry a centred- temperature slope:

\[ p(T, t) = \ell(T) + \frac{u(T) - \ell(T)}{1 + \exp\!\bigl[k(T) \cdot (\log_{10} t - \text{mid}(T))\bigr]} \]

with \(u, \ell, k, \text{mid}\) each modelled as \(\beta_0 + \beta_1 \cdot (T - \bar T)\). Default priors from bayesTLS::make_4pl_priors() are used. Samples are drawn with cmdstanr (2 chains × 2,000 iterations, half discarded as warm-up; adapt_delta = 0.9, max_treedepth = 12). Post-fit, \(z\) and \(\text{CT}_{\max,1\text{h}}\) are extracted with bayesTLS::extract_tdt(..., lethal = TRUE) using the rate-multiplier definition of \(T_{\text{crit}}\). We report the posterior median and 2.5 % / 97.5 % quantiles.

2 Performance metrics

For each estimator and each output quantity (\(z\), \(\text{CT}_{\max,1\text{h}}\)), each simulated dataset contributes one row with signed bias, squared bias (for RMSE aggregation), 95 % CI lower / upper, coverage indicator, and CI width. Per-cell summaries report:

  • Mean signed bias and Monte-Carlo SE of the bias
  • RMSE and Monte-Carlo SE
  • Empirical 95 % coverage rate and Monte-Carlo SE (binomial proportion)
  • Median 95 % CI width

These are reported twice per cell — once for all fits, once for fits that pass all joint-4PL convergence diagnostics (see Diagnostic checks).

2.1 Sample-size adequacy

Monte-Carlo standard error thresholds: 0.05 °C for bias, 0.015 for coverage. At \(N_{\text{sim}} = 1000\) (Scenarios 1–5) these thresholds are comfortably met for all method × quantity combinations under the baseline DGP. At \(N_{\text{sim}} = 500\) (Scenarios 6–8) the MCSE is \(\sqrt{2}\approx 1.41 \times\) larger; for the sensitivity sweeps this is an accepted trade-off for halving the per-cell compute time. The driver auto-checks both thresholds at every cell and prints a warning if either is exceeded.

3 Truth

For each simulated dataset, the true \(z\) and \(\text{CT}_{\max,1\text{h}}\) are computed analytically from the data-generating curve. The procedure: evaluate the true \(\log_{10} \text{LT}_{50}\) at every design temperature; fit a line through those values; read off \(z = -1 / \text{slope}\) and \(\text{CT}_{\max,1\text{h}} = (\log_{10} 60 - \text{intercept}) / \text{slope}\). For the baseline shape (only midpoint varying with \(T\)), \(\log_{10}\text{LT}_{50}(T)\) is exactly linear in \(T\) and the procedure is an analytical identity. For shape-perturbing scenarios, the fitted line is the linear projection of the (non-linear) true \(\log_{10}\text{LT}_{50}(T)\) curve onto the design grid — which is the target both estimators converge to, by construction of the OLS at Stage 2.

The implementation lives in compute_ols_truth() in scripts/sim_twostage_helpers.R and is checked for feasibility before any data is simulated: the function rejects DGPs whose \(u_T\), \(\ell_T\), or \(k_T\) leave \(0 < \ell < 0.5 < u < 1\) or \(k > 0\) at any design temperature.

4 Design grid

The default design grid is five assay temperatures \(T \in \{30, 32, 34, 36, 38\}\,^\circ\text{C}\) × six exposure durations \(t \in \{1, 5, 15, 45, 135, 405\}\) minutes — 30 cells per dataset. Scenario 8 explicitly varies this between the full grid and a sparse grid (three temperatures \(\{30, 34, 38\}\,^\circ\text{C}\) × four durations \(\{1, 15, 135, 405\}\) minutes — 12 cells per dataset).

Each (temperature × duration) cell is replicated by \(n_{\text{reps}}\) independent cups (the number varies by scenario). Each cup contains \(N\) individuals, \(N\) drawn from a discrete uniform on \(\{10, 11, \ldots, 20\}\) — capturing the modest, unbalanced replication typical of empirical TDT assays. So a Scenario-1 dataset with \(n_{\text{reps}} = 5\) contains \(5 \text{ T} \times 6 \text{ durations} \times 5 \text{ reps} = 150 \text{ cups}\), each with 10–20 individuals, for a total of roughly 1,500–3,000 individuals per simulated experiment.

Eight scenarios. Scenarios 1–2 establish equivalence and isolate the likelihood effect; Scenarios 3–5 introduce qualitatively different shape perturbations; Scenarios 6–8 are sensitivity sweeps that map the dose-response of the comparison against parameters varied within a scenario.

6 Diagnostic checks

Every joint-4PL fit logs:

Statistic Threshold
Maximum R̂ across parameters < 1.01
Minimum bulk ESS > 400
Minimum tail ESS > 400
Divergent transitions (post-warmup) = 0
Tree-depth saturations (post-warmup) = 0
BFMI (per chain) > 0.3
Combined all_pass flag all of the above

Per-cell summary reports: pass rate for each individual criterion, the overall all_pass rate, and median + worst-case of each statistic across the 500–1000 fits in the cell. Cells with all_pass rate < 95 % are flagged for review.

The bias / coverage summary is reported twice per cell:

  • All fits — every simulated dataset where the estimator produced finite point estimates. Reflects the realistic experience of running the recommended pipeline with default sampler settings.
  • Convergent fits only — restricted to fits where the joint 4PL satisfies all diagnostic criteria. If the two summaries diverge, pathological fits are biasing the headline numbers and sampler settings need to be revisited before the simulation is interpretable.

7 Reproducibility

The full set of per-simulation results — point estimates, CIs, all posterior draws of \(z\), \(\text{CT}_{\max,1\text{h}}\) and \(T_{\text{crit}}\) per dataset, and the full diagnostic row per fit — is stored as resume- safe per-sim RDS files under output/sim_twostage/raw/<scenario_label>/ and rolled up into summary_<label>.rds, meta_<label>.rds, draws_<label>.rds, and diffs_<label>.rds at the end of each cell.

The driver is scripts/sim_twostage_bias.R; the data-generating helpers and the three estimator wrappers live in scripts/sim_twostage_helpers.R. The queue dispatcher is scripts/run_sweep_queue.sh; it runs a pre-flight pass over every planned cell before any compute starts (see feedback_sim_preflight.md for what the pre-flight checks).

Parallelism uses a PSOCK cluster (parallel::makeCluster + parallel::parLapply) with five workers; a PSOCK cluster (rather than mclapply) is required because cmdstanr launches Stan executables via processx, which is not fork-safe under macOS.
