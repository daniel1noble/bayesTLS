1 Summary

A 1000-replicate Monte Carlo simulation comparing the joint Bayesian 4PL estimator against the classical two-stage TDT pipeline, on data generated under four data-generating processes (DGPs) and two replication budgets (3 or 5 cups per (T × duration) cell). The methods, DGP details, and the design are described in 2026-05-13-two-stage-bias-sim-methods.qmd.

3 Monte Carlo SE adequacy — is N = 1000 enough?

We pre-specified the following MCSE thresholds (set before running any simulations):

  • Bias MCSE < 0.05 °C
  • Coverage MCSE < 0.015 (i.e., the 95% interval on a coverage estimate is narrower than ±0.03)

Both thresholds are slack relative to the smallest effects we’d consider interpretable (≈ 0.1 °C in bias, ≈ 0.03 in coverage). Cells failing either threshold would need additional replicates.

Code 
mcse_tab <- summary_all |>
  mutate(bias_ok = mcse_bias < 0.05,
         cov_ok  = mcse_cov  < 0.015,
         both_ok = bias_ok & cov_ok) |>
  select(design, dgp, method, quantity,
         mcse_bias, bias_ok, mcse_cov, cov_ok, both_ok) |>
  arrange(design, dgp, method, quantity)
knitr::kable(mcse_tab, digits = 4)
Code 
n_total <- nrow(mcse_tab)
n_pass  <- sum(mcse_tab$both_ok)
worst_bias <- max(mcse_tab$mcse_bias)
worst_cov  <- max(mcse_tab$mcse_cov)

31 of 32 cells pass both thresholds at N = 1000. The largest observed bias MCSE is 0.0299 °C (threshold 0.05) and the largest coverage MCSE is 0.0152 (threshold 0.015). On these criteria, N = 1000 is sufficient for every cell.

4 What the data say — and what they do not

(i) Under baseline. Two-stage carries a positive bias of roughly 0.5 °C on \(z\) and a negative bias of roughly 0.5 °C on \(\mathrm{CTmax_{1hr}}\) at both designs. Joint 4PL bias is at most ≈ 0.08 °C on \(z\) and ≈ 0.06 °C on \(\mathrm{CTmax_{1hr}}\). Two-stage’s 95% Wald interval undercovers — 0.76 (n3) → 0.67 (n5) for \(\mathrm{CTmax_{1hr}}\), 0.87 (n3) → 0.82 (n5) for \(z\) — whereas the joint 4PL’s 95% credible interval is at nominal.

(ii) Coverage gets worse with more data for two-stage. This is visible in Figure 2 and Table 2: two-stage’s coverage on \(\mathrm{CTmax_{1hr}}\) drops from 0.76 at n3 to 0.67 at n5. The Wald interval shrinks faster than the bias does, so the (smaller) interval misses the (still-shifted) truth more often. The joint 4PL’s coverage stays at nominal across the same change in \(N\).

(iii) Bias deltas across DGPs are a truth-definition artefact, not a method response. For every DGP departure from baseline, the bias shift relative to baseline (same method, same quantity, same design) equals the OLS-truth shift between the two DGPs, to four decimal places:

Code 
b <- summary_all |>
  select(design, dgp, method, quantity, mean_bias) |>
  pivot_wider(names_from = dgp, values_from = mean_bias)

t_shift <- truths_df |>
  pivot_longer(c(z_true, ct_true),
               names_to = "quantity", values_to = "truth") |>
  mutate(quantity = ifelse(quantity == "z_true", "z", "CTmax_1hr")) |>
  pivot_wider(names_from = dgp, values_from = truth)

bias_delta <- b |>
  transmute(design, method, quantity,
            asym_u    = asym_u    - baseline,
            sym_ul    = sym_ul    - baseline,
            varying_k = varying_k - baseline)

truth_delta <- t_shift |>
  transmute(quantity,
            asym_u    = baseline - asym_u,
            sym_ul    = baseline - sym_ul,
            varying_k = baseline - varying_k)

cmp <- bias_delta |>
  pivot_longer(c(asym_u, sym_ul, varying_k),
               names_to = "dgp", values_to = "bias_shift") |>
  left_join(
    truth_delta |>
      pivot_longer(c(asym_u, sym_ul, varying_k),
                   names_to = "dgp", values_to = "truth_shift"),
    by = c("quantity", "dgp")
  ) |>
  arrange(design, method, quantity, dgp)
knitr::kable(cmp, digits = 4)

Mechanically: when the DGP changes (e.g. asym_u puts a temperature slope on \(u\)), the binomial counts on which both estimators operate differ from those under baseline, but the average point estimate produced by each method is essentially the same. Only the target moved. The variations we ran — u_beta1 = ±0.01, ell_beta1 = ±0.01, k_beta1 = +0.25 per °C — are too small to produce data the methods can distinguish from baseline.

What the present run therefore does not establish: how the two methods compare under temperature-varying \(u\), \(\ell\), or \(k\) that is large enough to actually shift the data. A follow-up run at substantially larger DGP magnitudes (e.g. u_beta1 = −0.05 per °C, k_beta1 = ±1.0 per °C) would be required to answer that question.

5 Open confounders (carried over from the baseline note)

  1. Prior sensitivity. Joint 4PL coverage and bias depend on the prior. We have not varied it.
  2. Implementation symmetry. Two-stage Stage-1 uses a 2-parameter logit GLM. A 4-parameter dose-response per temperature (e.g. drc::drm) would have a different per-T LT50 estimate; the current run benchmarks against the most common, simplest field implementation.
  3. MCMC pathology. Joint 4PL produced ≥ 95% of fits with zero divergences across all 8 cells, and no fit exceeded \(\hat R = 1.05\). We have not checked whether bias conditional on divergences = 0 differs from the marginal.
  4. Design coverage. Two design points (n3, n5) at one set of temperatures; no characterisation as a continuous function of design.
